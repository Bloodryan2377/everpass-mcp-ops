# Productivity stack — operational insights

Living notes. Update when a router pick was right, wrong, or missing.

## Defaults that compound

1. **Document skills > freestyle Office** — `pdf` / `docx` / `xlsx` / `pptx` preserve structure better than prompt-only generation.
2. **Repomix before multi-file dashboard work** — fewer missed deps than selective `@` when the tree is large.
3. **NotebookLM skill > paste** — if sources already live in a notebook, query in place; OneDrive remains canonical.
4. **Marketing skills for narrative; EverPass HARD rules for guardrails** — copywriting/cold-email/launch/ai-seo are accelerators, not permission to name individuals externally or send mail.
5. **ECC agents for review quality; EverPass skills for domain** — code-reviewer/security ≠ deal-term-sheet/dashboard.
6. **Many skills installed is cheap; many MCPs enabled is expensive** — keep MCP active set small.
7. **Flowise / Dify / Onyx are overflow valves** — not daily defaults. Claude Code + EverPass skills own the main path.
8. **Announce the router** — the one-line `Router: …` creates accountability and trains better future picks.
9. **Router miss → self-improve** — freestyling when a skill existed is a process bug, not a flex.
10. **HTML decks remain EverPass-native** — prefer `everpass-presentation` over pptx unless PowerPoint is explicitly required.
